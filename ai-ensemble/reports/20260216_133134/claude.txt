# AI 뉴스 수집기 리뷰

## 요약
`plan_request.md` 파일에 작성된 프로젝트 요구사항은 AI 기반 뉴스 수집 시스템의 **초기 컨셉**으로서 방향성은 명확하나, 실제 구현을 위해서는 **구체성이 부족**합니다. 기획 문서로서 필수 요소(사용자 스토리, 비기능 요구사항, 제약사항)가 누락되어 있으며, 기술 스택과 기능 설명이 너무 추상적입니다.

---

## 개선항목

### 1. **정확성(Correctness)** - 요구사항 명확화 필요

**문제점:**
- "RSS 피드 수집, 요약, 태깅"이 구체적으로 무엇을 의미하는지 불분명합니다.
  - 수집 대상 RSS 소스는? (고정 목록 vs 사용자 추가 가능)
  - 요약 방식은? (LLM 기반 vs 추출적 요약)
  - 태깅 기준은? (자동 분류 vs 사용자 정의 태그)

**해결 방안:**
```markdown
## 핵심 기능 (User Stories)
1. **RSS 수집**: 관리자가 설정한 N개 RSS 피드를 매 시간 크롤링하여 신규 기사를 수집
2. **AI 요약**: Claude API를 사용하여 기사 본문을 3줄 요약 생성 (원문 길이 > 500자)
3. **자동 태깅**: 기사 내용을 분석하여 `정치`, `경제`, `기술` 등 카테고리 자동 분류
4. **검색 및 필터**: 사용자가 태그/키워드로 기사 검색 가능
```

---

### 2. **스타일(Style)** - 기술 스택 선정 근거 부재

**문제점:**
- "Python, FastAPI, React"를 왜 선택했는지 이유가 없습니다.
- 대안 기술(예: Django vs FastAPI, Vue vs React)을 검토했는지 알 수 없습니다.

**해결 방안:**
```markdown
## 기술 스택 선정 이유
- **Backend (FastAPI)**: 비동기 RSS 크롤링에 적합, Pydantic 기반 타입 안전성
- **Frontend (React)**: 대시보드 UI 컴포넌트 재사용성 높음
- **AI (Claude API)**: 요약 품질 우수, 한국어 지원
- **DB (PostgreSQL + pgvector)**: 기사 유사도 검색을 위한 벡터 DB 필요
```

---

### 3. **보안(Security)** - 인증/권한 관리 누락

**문제점:**
- 누가 RSS 소스를 추가/삭제할 수 있는지 명시되지 않았습니다.
- API 엔드포인트에 대한 접근 제어 정책이 없습니다.

**해결 방안:**
```markdown
## 보안 요구사항
- **인증**: JWT 기반 로그인 (관리자/일반 사용자 구분)
- **권한**: 
  - 관리자만 RSS 소스 추가/삭제 가능
  - 일반 사용자는 읽기 전용
- **API Rate Limiting**: Cloudflare / FastAPI의 `slowapi` 적용
```

---

### 4. **성능(Performance)** - 확장성 고려 부족

**문제점:**
- RSS 피드가 100개 이상으로 늘어나면 크롤링 시간이 급증할 수 있습니다.
- 요약 생성 시 Claude API 호출 비용/속도 제한을 고려하지 않았습니다.

**해결 방안:**
```markdown
## 성능 최적화 전략
1. **비동기 크롤링**: `httpx.AsyncClient`로 병렬 요청 (동시 처리 20개)
2. **캐싱**: Redis를 사용하여 이미 수집한 기사는 재요약 방지
3. **백그라운드 작업**: Celery + RabbitMQ로 요약 작업 큐 처리
4. **페이지네이션**: 기사 목록 API는 50개 단위로 반환
```

---

### 5. **추가 누락 사항**

#### (1) 비기능 요구사항
```markdown
## 비기능 요구사항
- **응답 시간**: 기사 목록 API는 300ms 이내 응답
- **가용성**: 99.5% 업타임 (월 3.6시간 다운타임 허용)
- **확장성**: RSS 소스 1,000개까지 지원
```

#### (2) 데이터 스키마 힌트
```markdown
## 예상 DB 스키마 (초안)
- **rss_sources**: id, url, name, category, is_active
- **articles**: id, source_id, title, url, content, summary, published_at
- **tags**: id, name
- **article_tags**: article_id, tag_id (다대다 관계)
```

#### (3) API 명세 예시
```markdown
## 주요 API 엔드포인트
- `POST /api/sources` - RSS 소스 추가 (관리자 전용)
- `GET /api/articles?tag=기술&limit=50` - 기사 목록 조회
- `GET /api/articles/{id}` - 기사 상세 + 요약
- `POST /api/articles/{id}/summarize` - 수동 재요약 트리거
```
